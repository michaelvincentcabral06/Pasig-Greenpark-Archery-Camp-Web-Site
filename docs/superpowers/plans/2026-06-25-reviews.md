# Curated Google Reviews Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Turn the generic "What our archers say" block into curated Google-style review cards with an overall rating summary + "Read all on Google" link, all editable in admin.

**Architecture:** Extend the existing `testimonials` CONTENT array + add three section fields, in the single SuperConductor component in `index.html` (mirrored to `Pasig Greenpark Archery Camp.dc.html`). Content persists via the existing `persistContent`/`setContent` path — **no backend change, no deploy.**

**Tech Stack:** SuperConductor template, inline-styled markup, Playwright-core verification.

## Global Constraints

- **Mirror rule:** every `index.html` edit applied identically to `Pasig Greenpark Archery Camp.dc.html`; finish with `diff index.html "Pasig Greenpark Archery Camp.dc.html" && echo IDENTICAL`.
- **No backend change, no deploy.** Pure CONTENT JSON + frontend.
- **Preserve the visual language** — reuse existing card/section styling, brand colors (`#7fb43f` green, `#1b2a1f`/`#244232` dark, `#fffdf6` card, `#56664f` muted), fonts. Enhancement, not redesign.
- **SuperConductor:** NO JS ternaries inside style `{{ }}` interpolations — pre-compute booleans/strings on data objects; use `<sc-if>` for visibility.
- **Backward compatibility:** existing saved CONTENT (testimonials lacking `rating`/`date`, CONTENT lacking `reviews*`) must render via defaults (rating→5, date→'', reviewsRating→'5.0', count/url→'').
- **Verification:** Playwright-core driving the real DOM via the React-fiber `logic` instance (find element with `__reactFiber$…` key → walk `.return` to `stateNode.logic`).
